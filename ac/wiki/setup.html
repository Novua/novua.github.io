Install & bootstrap AnimationsCore

Follow these steps to deploy AnimationsCore on your server and wire the runtime API into your own plugins without shading or relocation headaches.

Installation Dependency management Bootstrapping the API

Installation & first run

Drop the plugin into your Paper / Spigot stack, then let it provision the data directories it needs.

  1. Place AnimationsCore.jar and MythicMobs.jar inside plugins/, then restart the server.
  2. On first boot AnimationsCore generates config.yml, player_preferences.yml, and catalogue folders such as joins/, leaves/, emotes/, teleport-ins/, and more.
  3. Optional: leave download-starter-pack: true (default) enabled to pull the Novua starter Mythic pack. The flag flips to false after the download completes.
  4. When the starter pack ZIP arrives, extract it and drop the labelled contents into the matching plugins/ModelEngine, plugins/MythicMobs, and plugins/AnimationsCore folders.
  5. After the server reloads ModelEngine's resource pack, merge it with your existing pack instead of replacing it outright.
  6. Grant staff the animcore.admin.* suite so they can reload configs, manage preferences, and trigger cinematics.
Tip

Animations can be disabled per-world through disabled_worlds in config.yml. Keep cinematic hubs pristine while leaving survival worlds untouched.

Resource pack safety

Do not delete player_shader when merging ModelEngine's generated resource pack with your own. Removing it breaks skin rendering for players.

Add AnimationsCore as a dependency

Pull the AnimationsCore API artefact from GitHub Packages so your own plugin can call into the runtime safely.

Maven

<repositories>
    <repository>
        <id>github</id>
        <url>https://maven.pkg.github.com/Novua/AnimationsCoreAPI</url>
    </repository>
</repositories>

<dependencies>
    <dependency>
        <groupId>net.novua</groupId>
        <artifactId>AnimationsCore</artifactId>
        <version>2.5.0</version>
        <scope>provided</scope>
    </dependency>
    <dependency>
        <groupId>net.novua</groupId>
        <artifactId>AnimationsCore</artifactId>
        <version>2.5.0</version>
        <classifier>sources</classifier>
        <scope>provided</scope>
        <optional>true</optional>
    </dependency>
    <dependency>
        <groupId>net.novua</groupId>
        <artifactId>AnimationsCore</artifactId>
        <version>2.5.0</version>
        <classifier>javadoc</classifier>
        <scope>provided</scope>
        <optional>true</optional>
    </dependency>
</dependencies>

Gradle (Kotlin DSL)

repositories {
    maven {
        url = uri("https://maven.pkg.github.com/Novua/AnimationsCoreAPI")
    }
}

dependencies {
    compileOnly("net.novua:AnimationsCore:2.5.0")
    compileOnly("net.novua:AnimationsCore:2.5.0:sources") // optional
    compileOnly("net.novua:AnimationsCore:2.5.0:javadoc") // optional
}

Bootstrap the API in your plugin

Depend on AnimationsCore without shading it into your own jar and guard against missing services at runtime.

  1. Add AnimationsCore to depend: or softdepend: inside plugin.yml.
  2. Mark the dependency as provided (Maven) or compileOnly (Gradle) so only the server ships the jar.
  3. Resolve the service inside onEnable() and disable your plugin gracefully if it is unavailable.
public final class ExamplePlugin extends JavaPlugin {
    private AnimationsCoreAPI animationsApi;

    @Override
    public void onEnable() {
        animationsApi = Bukkit.getServicesManager().load(AnimationsCoreAPI.class);
        if (animationsApi == null) {
            getLogger().severe("AnimationsCore API not available, disabling plugin");
            getServer().getPluginManager().disablePlugin(this);
            return;
        }

        registerCommands();
    }

    private void registerCommands() {
        Objects.requireNonNull(getCommand("welcome"))
            .setExecutor((sender, command, label, args) -> {
                if (!(sender instanceof Player player)) {
                    sender.sendMessage(ChatColor.RED + "Only players can use this.");
                    return true;
                }

                animationsApi.playAnimation(player, "join", "defaultjoin");
                animationsApi.openAnimationSelectionGui(player);
                return true;
            });
    }
}
Keep it modular

Place AnimationsCore-specific adapters under packages such as your.plugin.animations so the rest of your project depends on higher-level services instead of raw Bukkit lookups.
