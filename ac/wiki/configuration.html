Data folder layout

AnimationsCore provisions a consistent directory tree beneath plugins/AnimationsCore/. Each category stores MythicMob bindings and GUI metadata under animations keys.

Path Purpose
player_preferences.yml Per-player animation selections grouped by type.
joins/joins.yml, leaves/leaves.yml, deaths/deaths.yml, respawns/respawns.yml Lifecycle animation catalogues. Keys underneath animations. map to MythicMob IDs.
afks/afks.yml AFK cinematics, including transition timings and permission gates.
guis/guis.yml Layout and copy for the in-game selector GUI.
teleport-ins/teleports.yml, teleport-outs/teleports.yml Teleport entry / exit cinematics triggered by monitored commands.
emotes/emotes.yml Emote catalogue with duration, looping, and traversal metadata.
bans/bans.yml Ban cinematics executed alongside moderation commands.
kills/kills.yml Reserved for kill-event cinematics.
customs/customs.yml Manual-only animations for scripted cutscenes or one-off events.

Use the following template to create new entries. Keep the key names consistent across YAML files, GUI slots, and API calls.

animations:
  cinematic_a:
    mm-id: NovuaJoinAnimation
    permissions:
      - animcore.join.cinematic_a
    duration: 200
    gui:
      name: "&aNovua Arrival"
      material: ENDER_EYE
      slot: 10
  cinematic_b:
    mm-id: NovuaLeaveAnimation
    permissions: []
    duration: 120
    gui:
      name: "&bWave Goodbye"
      material: BLAZE_ROD
      slot: 11

Configuration reference

Core options from config.yml, summarised with defaults and behaviour notes.

Key Default Description
permission-message "You don't have permission to use this command." Fallback message for admin commands without bespoke feedback.
debug-mode false Verbose logging for Mythic spawns, AFK transitions, and spawn validation.
defaultanimations false Automatically apply the default* keys (join, leave, etc.) to first-time players.
defaultjoin, defaultleave, defaultdeath, ... Empty Choose the animation key granted when defaultanimations is enabled.
database.* Disabled / localhost Mirror player_preferences.yml into MySQL. Toggle per API call using syncDatabase.
spawn-point.* Disabled Stores coordinates for /animcore spawn. Populated through /animcore setspawn.
TeleportAnimations.enabled true Turns teleport cinematics on or off globally.
TeleportAnimations.listen-for 160 Number of ticks to watch for the teleport completion after a trigger command is used.
TeleportAnimations.triggers ["/tp", "/teleport"] Command list that starts the teleport cinematic handshake.
Camera.* enabled: true, transition: true, forwards: 10, height: 5 Camera offsets applied to join cinematics.
trigger-on-join-event true Start join cinematics immediately or wait for the resource pack event.
JoinAnimDelay 5 Delay (ticks) before playing the join animation after the trigger.
AFKTimeout 6000 Idle ticks (5 minutes) before AFK detection fires.
disabled_worlds [] Worlds where AnimationsCore refuses to spawn MythicMobs.
download-starter-pack true Automatically download the Novua starter pack on next boot (flag resets after success).
AutoHideLocked true Hide GUI entries if the viewer lacks the listed permissions.
Performance hint

Use animation-triggers.* in config.yml to disable listeners for categories you do not use. Skipping unused hooks trims event noise on larger servers.
