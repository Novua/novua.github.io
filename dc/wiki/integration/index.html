MythicMobs, ModelEngine, and QTE Integration

MythicMobs

  • spawnmob... instructions spawn MythicMobs and register aliases in sequence context.
  • clearmobs cleans up sequence-tracked spawned mobs.
  • Camera anchor/look sources can point at these aliases for mob-relative shots.

ModelEngine bone cameras (full flow)

  1. Have a ModelEngine model with a camera bone (common naming: cam_<instance>).
  2. Spawn or attach model instance:
    • camspawn(instance,modelId[,bone][,animation])
    • camattach(instance,mobAlias[,bone][,animation])
  3. Activate perspective: camplay(instance).
  4. When finished, cleanup: camclear(instance).
script:
  - spawnmob{mob=BossMob;id=boss;client=false}
  - camattach(bosscam,boss,cam_bosscam,intro)
  - camplay(bosscam)
  - delay 80
  - camclear(bosscam)

If camera facing looks offset, tune model_engine.camera_rotation.yaw_offset and pitch_offset in config.yml.

QTE branching

  • Trigger QTE in a sequence with qte(...).
  • Definition can automatically route to success_sequence / fail_sequence.
  • Use sendqteresult(alias) when you need QTE outcome bound to specific mob context for downstream mechanics.

Targeter context

When a sequence is launched from Mythic mechanics, DirectorsCut can resolve target entity/location context for source lookups and relative camera calculations.
