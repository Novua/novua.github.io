MythicMobs Integration

When MythicMobs is installed, DirectorsCut registers a mechanic named directorscut (aliases: directors_cut, dcscene) that can trigger sequences for a specific player.

DirectorsCut mechanic examples

Skills:
  - directorscut{sequence=drawbridge-openclose;player=<trigger.name>} @trigger
  - directorscut{sequence=<trigger.var.scene>;player=<trigger.name>} @trigger
  - directorscut{sequence=drawbridge-openclose} @trigger

Both sequence= and player= support Mythic variable placeholders. If player= cannot be resolved, DirectorsCut now attempts fallback context in this order: explicit player expression, targeted player, trigger player, then caster player.

Using spawnmob from sequence scripts

script:
  - spawnmob ZombieKnight
  - spawnmob{mob=ZombieKnight;client=true}
  • client=false (or omitted) keeps MythicMob spawns shared/global.
  • client=true scopes visibility so only the sequence viewer sees that spawned mob.

Targeter context

During this skill, the entity targeted by MythicMobs becomes the mythicmobs_targeter. Cameras can anchor or look at it with source: mythicmobs_targeter.
