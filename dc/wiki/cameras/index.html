Create Cameras

Cameras are stored as YAML entries inside cameras/*.yml. Each file contains a cameras map of camera IDs and their options.

Core camera fields

FieldPurpose
labelFriendly display name (defaults to the camera ID).
anchorDefines the base location. Supports player, entity, or absolute.
look_atOptional target to face. Use type: none to keep stored yaw/pitch.
positioningDefines offsets (relative, anchored, or absolute) and yaw/pitch offsets.
lock_playerLock movement or look input while the camera is active.

Example camera

cameras:
  village_intro:
    label: Village Intro
    anchor: { type: player, source: self }
    look_at: { type: entity, source: interacted }
    positioning: { mode: relative, distance: 3.0, yaw_offset: 30, y_offset: 1.0, pitch: -10 }
    lock_player: { move: true, look: false }

In-game camera creator

  1. Run /directorscut creator (requires the directorscut.creator permission).
  2. Move to the desired camera location.
  3. Press F (swap offhand) to save.
  4. The camera YAML is written to plugins/DirectorsCut/create/generated.

For every supported camera option, see the camera settings quick reference.
