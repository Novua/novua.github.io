Camera Settings Quick Reference

This page summarises the options available when defining cameras for DirectorsCut. Each camera entry lives inside a YAML file under DirectorsCut/examples/cameras (or your plugin data folder at runtime). The new camera_options_showcase.yml example demonstrates every combination below.

File-Level Keys

KeyDescription
pack Optional label shared by all cameras declared in the file. Packs can inform resource pack swaps or GUI grouping.
cameras Map of camera IDs to their settings. Each key becomes the internal camera identifier. Duplicate IDs overwrite earlier ones.

Per-Camera Settings

label

Friendly name presented to players. Defaults to the camera ID when omitted.

anchor

Defines the base location that positioning calculations stem from.

  • type — player, entity, or absolute. Defaults to entity.
  • source (for player/entity) — choose self (the viewing player), interacted (the entity that triggered the sequence), or mythicmobs:<id> (any mob spawned earlier in the sequence).
  • absolute block (for type: absolute) — supply world, x, y, z, yaw, pitch to lock the camera to a fixed position.

look_at

Optional. Uses the same structure as anchor. When provided, the camera rotates to face the resolved target plus any configured pitch offset.

  • Use type: none (or the legacy alias relative) to keep the stored yaw and pitch without tracking a player or entity.
  • When no look target can be resolved, the final rotation depends on the positioning mode (see below).

positioning

Controls where the camera sits relative to the anchor.

  • mode — relative (default), anchored, or absolute.
  • relative mode — offsets rotate with the anchor's yaw.
    • distance — blocks forward/back from the anchor facing direction (negative values move behind).
    • yaw_offset — degrees to rotate the offset arc; also nudges final yaw if no look target is found.
    • y_offset — vertical offset from the anchor.
    • pitch — pitch offset applied after look-at adjustment, or as absolute pitch if no target is resolved (including look_at: { type: none }).
  • anchored mode — offsets are treated as world-space differences from the anchor location. The stored yaw_offset and pitch are used verbatim when there is no look target, making the preview match what the creator recorded.
  • Absolute mode — supply an absolute block with world, coordinates, yaw, and pitch. The camera always spawns at those values before any look-at correction.

The in-game creator saves new cameras in anchored mode so the test preview and live playback line up exactly.

Note: Cameras are purely positional. Transition timing, glide lengths, and delays are configured inside sequence scripts.

lock_player

Optional block controlling player input while the camera is active. Omit to leave movement/look unlocked unless the sequence globally freezes players.

  • move — true disables WASD input.
  • look — true disables mouse look.

Complete Example

The snippet below is lifted from DirectorsCut/examples/cameras/camera_options_showcase.yml and combines each feature.

pack: camera_options_pack

cameras:
  player_self_relative:
    label: Player Self Relative
    anchor: { type: player, source: self }
    look_at: { type: entity, source: interacted }
    positioning: { mode: relative, distance: 2.0, yaw_offset: 25, pitch: -15, y_offset: 1.2 }
    lock_player: { move: true, look: false }

  absolute_showcase:
    label: Absolute Showcase
    anchor:
      type: absolute
      absolute: { world: world, x: 120.5, y: 85.0, z: -10.5, yaw: 135.0, pitch: -25.0 }
    look_at: { type: player, source: self }
    positioning:
      mode: absolute
      absolute: { world: world, x: 118.0, y: 90.0, z: -8.0, yaw: 160.0, pitch: -30.0 }
    lock_player: { move: false, look: false }

Refer to the YAML file itself for additional variations using interacted anchors, MythicMobs sources, and absolute look targets.

Sequence Script Instructions

Sequences are plain-text scripts located in DirectorsCut/sequences. Each non-empty line inside the main script list is parsed into one of the instructions below. An optional endscript list runs after the sequence ends and accepts the same instruction syntax—perfect for cleanup logic.

InstructionPurposeExample
static(id) Teleport the viewer to the specified camera instantly. Use delay instructions to hold the shot, and glide for eased transitions. static(amonde_skeleton_boss-intro1)
glide(from,to,duration) Ease between two cameras over duration ticks. The third value is the authoritative animation length—camera files ignore the legacy duration_ticks field. glide(intro1,intro2,40)
delay <ticks> Pause script execution for the specified number of ticks (20 ticks = 1 second). delay 60
spawnmob <mythic_id> Spawn a MythicMobs entity and register it for later camera targets via look_at: { source: mythicmobs:<id> }. spawnmob skeleton_boss
command(c="...",asplayer=<bool>,asop=<bool>) Run any Bukkit command. Set asplayer to true to run as the viewer, and asop to true to elevate temporarily. command(c="mm m spawn skeleton_boss 1 void,280,101,58,139.5,0",asplayer=false)
showtitle(fadeIn,stay,fadeOut,"text") Display a title to the viewer. Use showsubtitle or showactionbar with the same signature for other overlays. showtitle(20,60,20,"&6Beware!")
clearmobs Despawn any MythicMobs spawned by spawnmob earlier in the sequence. clearmobs
end Terminate the sequence immediately (the same as an endscript marker). Useful for optional cutaways. end
conditionend End the current conditional branch while allowing the main script to continue. conditionend

Example clean-up block that despawns MythicMobs and returns the player to survival mode:

endscript:
  - clearmobs
  - command(c="gamemode survival %player%",asplayer=false)
  - delay 5
  - command(c="title %player% clear",asplayer=false)

Lines beginning with # are treated as comments. Escape quotes in command and title strings with \", and use \n for new lines where supported.
